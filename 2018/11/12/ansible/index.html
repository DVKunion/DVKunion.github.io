0x00- - Ansible 简介

Ansible 是一个基于 python 开发、ssh 协议的自动化运维的工具。集成了众多的运维工具的优点，能够在需要在众多机器上部署代码和服务时进行批量处理。Ansible 能够为我们进行一些批量、重复的工作。

相比于其他过往的运维管理工具，其优势主要在于其自身属于一个“轻量级”工具，采用现有的SSH安全通讯机制，具体操作层面将运维人员从复杂的代码逻辑上解放出来，以简单易学的语法格式实现复杂的管理需求。

Ansible优点：

  • 安装部署简单
  • 基于SSH对客户端进行管理
  • 无需运行守护进程
  • 日志集中存储
  • 简单易用，功能强大

这里进行一个形象的比喻来帮助理解Ansible的优点：

  • SSH : 框架协议——类似房子的图纸。
  • OpenSSH：手工土瓦房——命令全靠自己手动输入，所有的一切都需要自己本身操作。
  • XShell：农村二层小洋房——只有一个好看的外形，其本质与SSH等并无过多差异。
  • Paramiko ：农村现代化别墅——实现可编程控制操作——但是需要自己处理逻辑关系。
  • Fabric：城市小公寓——需要做的事情更少——但依旧存在
  • Ansible：超级别墅——所有你需要做的都有专职的佣人存在。
  • Ansible中的两大模块:
    • ——-ansible秘书：CMDB—-帮助user创建Inventory。
    • ——-playbooks : 相当于一张时间表，告诉助手每天需要做些什么工作、工作的顺序等等。

0x02- - Ansible 运行&模块

Ansible 运行

Ansible 的运行模式有两种：Ad-Hoc 和 Playbook

  1. Ad-Hoc
    ad-hoc 命令是一种可以快速输入的，无需要保存的命令。类似命令行中的命令。临时命令适用于下面类似的场景，如需要临时重启一台或几台主机，只需要ansible的一行命令即可，而不必编写一个专门的文件来完成这个工作。

简单的来说，ad-hoc 类似于 xshell 等工具、依旧是需要管理者手动输入命令，对命令进行单独操作。常用于一些功能简单、步骤数目少的操作，对于这样的操作写一个Playbook脚本反而会显得笨拙繁琐。

  1. Playbook
    Playbook是一系列任务的集合，称为“剧本”，每一项任务就是“剧本”中的一出“戏”。

在简介中，我把Playbook 比作时间表。所谓 Playbook 就是一张写好的命令清单，详细的描述了每个任务的执行顺序， 交给Ansible各个模块去执行。
